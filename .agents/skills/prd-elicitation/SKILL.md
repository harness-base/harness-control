# 需求产出（用户故事 → PRD → 原型）

`dev` 管"实现需求"，本 skill 管它的**上游——产出需求**：分阶段把"做什么"想清楚（用户故事 → PRD → 可选原型），**每阶段用户确认**，再由 `dev` 接力"怎么做"。**松耦合**：实现体系不强依赖 PRD 存在，有则平滑衔接（一份 PRD 派生 1..N 个 feature 包）。流程依据见 ADR-0007。

## 何时用 / 何时不用
- 用：从模糊想法或现有系统理出需求；要用户故事 / PRD / 原型；方向、页面、流程得先想清楚再动手。
- 不用：需求已清晰、只是开发切片（直接走 `dev`）；纯控制面改动。

## 贯穿全程的约束（不是某一步）
- **不静默假设**：任何信息不足，**去查事实源**（仓内代码 / 资产 / 文档）或**摆出来问用户确认**，绝不拿"已有信息 + 想要的结果"硬编。
- **两套优先级**（信息冲突时按此裁）：
  - **真相源**（什么是真的）：① 用户当场直说 > ② 仓内现有代码 / 资产 / 现状 > ③ 用户提交的信息源（目录 / 文档 / 图）。
  - **需求源**（要做成什么）：① 用户当场直说 > ② 用户提交的信息源 > ③ 现状。
- **前置**（不算正式步骤）：用户指定 / 提交了信息源，先读它再开工。

## 工作流（5 步，每步多轮沟通 + 用户确认门）
1. **需求澄清**：分轮引导对话——用户与 JTBD → 页面与流程 → 数据 → 四态（空 / 加载 / 错误 / 成功）+ 边界 → 验收目标 → 非目标。关键抉择用 `AskUserQuestion`，开放探索用对话，迭代到画面够清。
2. **产出用户故事**：套 `templates/user-story.md`，落**独立产物** `docs/prds/<id>/user-stories.md`（每条 `US-NN`）。**用户多轮审核 → approved 才进下一步**。它是后续 PRD / 原型 / 评审统一对齐"需求"的**事实视角**。
3. **产出 PRD**：套 `templates/prd.md`，落 `docs/prds/<id>/prd.md`，登记 `index.yaml`。**须符合已确认用户故事**，带**功能点清单 + `US↔FP↔正文` 映射**（三级双向覆盖，软，目标 100%）。写法照 `references/prd-writing.md`（人类可读、业务逻辑清晰、旧场景简 / 繁分别处理）。**用户多轮审核确认**。
4. **（可选）产出原型**：可点的 HTML，落 `docs/prds/<id>/prototype/`，带 UI 属性、mock 数据无后端；**与现有前端内容 / 风格一致（不复刻还原，见 ADR-0003）**做需求相关原型。无界面 / 生成不了 → 跳过；能生成 → **先问用户要不要**；要则**多轮审核确认**。
5. **对抗式完整性评审（loop engineering）**：PRD + 原型对标准自查（覆盖映射齐？验收可观测？四态齐？流程闭合？原型点得通？假设都确认？），必要时多 agent 对抗证伪，**审 → 修 → 再审，到没问题为止**。

完成后可派生 feature 包（由 `dev` 实现，不强制）。

## 硬规则
- 引导式、多轮、收敛；**不静默假设**（贯穿，见上）。
- **用户故事 = 独立产物 + 对齐锚**：**先于 PRD 产出**、approved 才进 PRD；PRD / 原型 / 评审都回它对齐。
- 验收目标**可观测可验证**，不是"做完了"。
- PRD 带**功能点清单 + 覆盖映射**（软，目标 100%）。
- 原型**能点通主流程**；项目有现有前端则与之内容 / 风格一致。
- 产物**独立存放**于 `docs/prds/<id>/`，与实现体系**松耦合**。

## 门禁（rule-0010 / 考题 013）
- 产出 PRD 须满足 **rule-0010**（PRD 产出标准：验收可观测、范围闭合、四态齐、原型可点通、假设确认、可追溯、登记不漂移）。
- 结构（登记一致 + `prd.md` 必备章节）由 `scripts/prds-audit.sh` 机器校验，进 `make verify`。
- 收尾跑 eval **考题 013**（用户故事 / PRD / 原型 完整性与可观测性）。
- **不强制 PRD 必须存在**（松耦合，ADR-0003）——门禁只在产出时适用。

## 演进（rule-0007）
流程 / 标准变化时回顾本 skill（含 `references/prd-writing.md`）；改完同步 `version` / `last_reviewed`，跑 `bash scripts/skills-index.sh`。
