# eval 010 — 任务收尾综合评审（rule-0005）

## 评什么
命中 rule-0005 判据（多步改产物 / 写了 ADR / 动业务代码 / 关键决策点）的任务收尾整体质量。综合套用相关考题（002、003、011、012）。

## 检查清单
- 验证：结论分类如实吗（002)、有真实证据吗（003）。
- 断言：e2e/验收断言锚定产出方证据了吗（012）——警惕访问日志回显造成的假阳性。
- skill：大改回顾了吗（011）。
- 证据结构：命令 / 时间 / 环境 / 结果 / 分类 / case id 齐吗。
- 结论时效：本轮动过的产物，**最后一版**过对抗评审了吗——评审通过后又改 → 旧结论作废。免评理由**去 `tasks/todo.md` 当前任务的 Review 段找那一行；找不到 = 未免评**（不接受"应该属于琐碎改动"这类口头解释）。机检绿 / eval 绿都不替代对抗评审。**同处再查一条：`tasks/todo.md` 当前任务 Review 段里有没有用户明示验收通过的原话 + 日期——找不到 = 流程未结束。** 口径见 `docs/harness/adversarial-review.md`「流程什么时候算结束 · 改动后何时必须重新评审」。

## 输出
逐项 verdict + 综合分档（green / yellow / red）+ 给用户的一句话提示。
